Intraday Return and Volatility Modeling

Introduction¶

This notebook looks at intraday market data for a selected ticker and applies two time-series models: ARIMA for return dynamics and GARCH for volatility dynamics. The main goal is to see how well these models can describe short-term behavior in high-frequency financial data and how useful they are for out-of-sample forecasting.

Instead of working with raw prices directly, the analysis focuses on resampled intraday data and log returns, which are more appropriate for time-series modeling. The notebook also compares model forecasts across training, validation, and test periods to keep the evaluation realistic.

Key Features¶

  • Filters raw trade data to a chosen ticker and date range

  • Resamples irregular observations to a fixed frequency

  • Computes log returns within each trading session

  • Uses ARIMA to model return behavior over time

  • Uses GARCH-family models to model time-varying volatility

  • Includes diagnostic plots and residual checks

  • Evaluates models using out-of-sample validation and test sets

  • Compares GARCH forecasts with a simple constant-volatility baseline

Methodology¶

The notebook starts by cleaning and restructuring the raw intraday data. Since trades do not occur at perfectly regular time intervals, prices are resampled onto a fixed grid. This makes the data easier to model and compare across sessions. Returns are computed within each session so that overnight gaps are not treated as normal intraday movements.

The data is then split chronologically into training, validation, and test sets. (70/15/15) This is important because time-series models should be evaluated in the same order the data actually occurs, rather than with random splits.

Modeling Approach¶

Two different modeling approaches are used in this notebook:

ARIMA¶

The ARIMA model is used to study patterns in intraday log returns. It is mainly used here to check whether returns show serial structure that can be modeled and forecasted over short horizons. Diagnostic tools such as ACF, PACF, and residual checks are used to evaluate whether the model is a reasonable fit.

GARCH¶

The GARCH model is used to study time-varying volatility in intraday log returns. This is useful because financial returns often show volatility clustering, where large moves tend to be followed by large moves. The notebook tests whether a dynamic volatility model can do better than a simple constant-volatility benchmark.

Overall Goal¶

Overall, this notebook is meant to compare how ARIMA and GARCH handle different parts of intraday financial data. ARIMA focuses more on the behavior of returns themselves, while GARCH focuses on changes in volatility. Together, they give a broader view of short-term market dynamics and forecasting performance.

Data Description¶

This dataset contains intraday market data for a selected ticker over a chosen date range. Each record represents a time bucket within a trading session and includes both the observed filled price and the corresponding log return, when available.

Variables¶

  • symbol: the ticker being analyzed

  • session_date: the trading date

  • second_bucket: intraday timestamp for each observation

  • filled_price: the observed trade or fill price at that time

  • log_return: the logarithmic return associated with the price movement

Processing steps¶

The data is:

  1. Filtered to a single ticker and date range

  2. Sorted by session_date and second_bucket

  3. Downsampled to 10-second intervals

  4. Split into two series:

    • price observations where filled_price is not null

    • return observations where log_return is not null

Visualizations - see graphs for prices, log returns for each future¶

Price plot¶

The first chart shows filled_price over time for each trading session.
Each line represents one session_date, making it easy to compare intraday price behavior across days.

Log return plot¶

The second chart shows log_return over time for each trading session.
This highlights short-term price changes and makes intraday volatility patterns easier to see.

Summary¶

Together, these plots provide a view of:

  • how the ticker price evolves during the trading day

  • how returns fluctuate at a high frequency

  • how intraday behavior differs across sessions

ARIMA analysis of intraday log returns¶

This analysis applies an ARIMA model to intraday log returns for a single futures ticker. The objective is to evaluate whether a simple univariate time-series model can capture short-horizon dependence in returns and produce reasonable out-of-sample forecasts.

Data preparation¶

The raw data is first filtered to a single ticker over a specified date range. From the available intraday observations, the analysis keeps the trading session date, the intraday timestamp, and the observed filled price. Because the original data is irregularly spaced, prices are resampled onto a fixed interval grid (for example, every 10 seconds) within each trading session.

This preprocessing step is important because ARIMA models assume observations occur at regular time intervals. Missing values created during resampling are forward-filled within each session, and sessions with too few usable observations are excluded from the analysis.

Modeling choice¶

Rather than modeling price levels directly, the analysis models log returns, computed from the resampled price series as the first difference of log prices within each session. This is a more appropriate target for time-series modeling because price levels are typically non-stationary, while returns are often more stable over time and more suitable for ARIMA-type methods.

The model is therefore estimated on a single return series constructed from the cleaned intraday data.

Train, validation, and test design¶

To preserve the time-series structure, the data is split in chronological order into three segments:

  • 70% training

  • 15% validation

  • 15% test

This avoids look-ahead bias and ensures that forecasting performance is evaluated in a way that reflects a genuine prediction setting.

Model diagnostics¶

Several standard diagnostic tools are used to assess whether the ARIMA specification is reasonable:

  • a plot of the full log-return series

  • a plot of the training return series

  • the autocorrelation function (ACF)

  • the partial autocorrelation function (PACF)

  • residual plots after estimation

  • the Ljung–Box test for remaining autocorrelation

Together, these diagnostics help determine whether the model captures the serial dependence in the training data and whether residual structure remains after fitting.

Forecast evaluation¶

The ARIMA model is first fit on the training set and used to forecast the validation period. Forecast accuracy is measured using:

  • MAE (Mean Absolute Error)

  • RMSE (Root Mean Squared Error)

After this initial evaluation, the model is refit using the combined training and validation data, and a second forecast is generated for the test set. This provides a final out-of-sample assessment of predictive performance.

Interpretation¶

At a high level, this exercise tests whether a relatively simple ARIMA specification can explain and forecast intraday return dynamics for a single ticker. The focus is not on building a fully optimized forecasting system, but rather on applying a standard statistical time-series model in a disciplined way: regularizing the data, modeling returns instead of prices, preserving chronological splits, and evaluating performance on unseen observations.

Limitation¶

One important limitation is that, although returns are computed separately within each trading session, the final modeling series combines all sessions into a single sequence. As a result, the ARIMA model does not explicitly account for market-open effects, overnight information, or other session-specific structural changes.

from pyspark.sql import functions as F
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

from statsmodels.tsa.arima.model import ARIMA
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
from statsmodels.stats.diagnostic import acorr_ljungbox

def run_arima_one_ticker(
    mbo_spark_df,
    ticker="CL.c.0",
    start_date="2025-11-18",
    end_date="2026-01-20",
    freq="10s",
    order=(1, 0, 1),
    single_session_only=None,
    min_points_per_session=30
):
    df = (
        mbo_spark_df
        .filter(
            (F.col("symbol") == ticker) &
            (F.col("session_date") >= F.lit(start_date)) &
            (F.col("session_date") <= F.lit(end_date))
        )
        .select("session_date", "second_bucket", "filled_price")
        .where(F.col("second_bucket").isNotNull())
        .orderBy("session_date", "second_bucket")
    )

    if single_session_only is not None:
        df = df.filter(F.col("session_date") == F.lit(single_session_only))

    pdf = df.toPandas()

    if pdf.empty:
        raise ValueError(f"No data for {ticker} between {start_date} and {end_date}")

    pdf["session_date"] = pd.to_datetime(pdf["session_date"])
    pdf["second_bucket"] = pd.to_datetime(pdf["second_bucket"])
    pdf = pdf.sort_values(["session_date", "second_bucket"]).reset_index(drop=True)

    print(f"{ticker} | rows: {len(pdf)}")
    print("Sessions:", sorted(pdf["session_date"].dt.strftime("%Y-%m-%d").unique()))

    # Resample per session
    session_frames = []

    for session, g in pdf.groupby("session_date"):
        g = g.sort_values("second_bucket").dropna(subset=["filled_price"])

        if g.empty:
            continue

        g = g.set_index("second_bucket")

        s = g["filled_price"].resample(freq).last()
        s = s.ffill().dropna()

        if len(s) < min_points_per_session:
            continue

        tmp = s.to_frame("filled_price").reset_index()
        tmp["session_date"] = pd.to_datetime(session)
        session_frames.append(tmp)

    if not session_frames:
        raise ValueError("No sessions after resampling")

    ts_df = pd.concat(session_frames, ignore_index=True)
    ts_df = ts_df.sort_values(["session_date", "second_bucket"]).reset_index(drop=True)

    # Build log returns within each session
    ts_df["log_price"] = np.log(ts_df["filled_price"])
    ts_df["log_return"] = ts_df.groupby("session_date")["log_price"].diff()

    model_df = ts_df.dropna(subset=["log_return"]).copy()

    series = model_df.set_index("second_bucket")["log_return"]
    series = series[~series.index.duplicated()]

    if len(series) < 100:
        raise ValueError("No data")

    print(f"Resampled points: {len(series)}")

    # Train / val / test split 70/15/15
    n = len(series)
    train_end = int(n * 0.70)
    val_end = int(n * 0.85)

    train = series.iloc[:train_end]
    val = series.iloc[train_end:val_end]
    test = series.iloc[val_end:]

    print(f"Train: {len(train)} | Val: {len(val)} | Test: {len(test)}")

    # Plots
    plt.figure(figsize=(14, 4))
    plt.plot(series, linewidth=1)
    plt.title(f"{ticker} log return")
    plt.grid(alpha=0.3)
    plt.show()

    plt.figure(figsize=(14, 4))
    plt.plot(train, linewidth=1)
    plt.title("Training log return")
    plt.grid(alpha=0.3)
    plt.show()

    plot_acf(train.dropna(), lags=50)
    plt.show()

    plot_pacf(train.dropna(), lags=50, method="ywm")
    plt.show()

    # Fit on training set
    model_train = ARIMA(train, order=order)
    results_train = model_train.fit()

    print(results_train.summary())

    # Residuals
    resid = results_train.resid.dropna()

    plt.figure(figsize=(14, 4))
    plt.plot(resid, linewidth=1)
    plt.title("Residuals (train)")
    plt.grid(alpha=0.3)
    plt.show()

    plot_acf(resid, lags=50)
    plt.show()

    lb = acorr_ljungbox(resid, lags=[10, 20, 30], return_df=True)
    print("\nLjung-Box:\n", lb)

    # Validation forecast
    val_forecast = results_train.get_forecast(steps=len(val))
    val_pred = val_forecast.predicted_mean
    val_ci = val_forecast.conf_int()

    val_pred.index = val.index
    val_ci.index = val.index

    val_mae = np.mean(np.abs(val - val_pred))
    val_rmse = np.sqrt(np.mean((val - val_pred) ** 2))

    plt.figure(figsize=(14, 5))
    plt.plot(train, label="Train")
    plt.plot(val, label="Val actual")
    plt.plot(val_pred, label="Val pred")

    plt.fill_between(
        val_ci.index,
        val_ci.iloc[:, 0],
        val_ci.iloc[:, 1],
        alpha=0.2
    )

    plt.legend()
    plt.title("Validation forecast: log returns")
    plt.grid(alpha=0.3)
    plt.show()

    print(f"\nVal MAE: {val_mae:.6f} | RMSE: {val_rmse:.6f}")

    # Refit on training + validation
    train_val = series.iloc[:val_end]
    results_train_val = ARIMA(train_val, order=order).fit()

    # Test forecast
    test_forecast = results_train_val.get_forecast(steps=len(test))
    test_pred = test_forecast.predicted_mean
    test_ci = test_forecast.conf_int()

    test_pred.index = test.index
    test_ci.index = test.index

    test_mae = np.mean(np.abs(test - test_pred))
    test_rmse = np.sqrt(np.mean((test - test_pred) ** 2))

    plt.figure(figsize=(14, 5))
    plt.plot(train_val, label="Train+Val")
    plt.plot(test, label="Test actual")
    plt.plot(test_pred, label="Test pred")

    plt.fill_between(
        test_ci.index,
        test_ci.iloc[:, 0],
        test_ci.iloc[:, 1],
        alpha=0.2
    )

    plt.legend()
    plt.title("Test forecast: log returns")
    plt.grid(alpha=0.3)
    plt.show()

    print(f"\nTest MAE: {test_mae:.6f} | RMSE: {test_rmse:.6f}")

    return {
        "ticker": ticker,
        "order": order,

GARCH analysis¶

This part of the project uses a GARCH model to study intraday log returns for each future. The goal is not to predict whether returns go up or down, but to model how volatility changes over time. GARCH makes sense here because financial returns often show volatility clustering, where big moves tend to be followed by big moves and quiet periods tend to stay quiet.

First, the raw trade data is filtered to one ticker and a chosen date range. The prices are then resampled to a fixed time interval so the data is on a regular grid. Returns are computed within each trading session, which is important because it avoids creating fake overnight returns between sessions.

A few important modeling choices:

The model uses log returns instead of prices, since GARCH is meant for return volatility, not price levels. The data is split in time order into training, validation, and test sets, which makes the forecasting setup more realistic. The code also allows different volatility models, like standard GARCH, EGARCH, and GJR-GARCH, plus either normal or Student-(t) errors.

To check whether GARCH is appropriate, we look at the ACF of returns and the ACF of squared returns. Squared returns are especially important because autocorrelation there suggests volatility clustering, which is exactly what GARCH is designed to capture. After fitting the model, we check standardized residuals, their ACF, the ACF of squared standardized residuals, and Ljung–Box tests to see whether the model has removed most of the remaining structure.

For forecasting, we used rolling one-step-ahead forecasts. The model is fit on the training data, evaluated on validation, then refit on train plus validation and tested on the final test set. Forecasted volatility is compared to a realized-volatility proxy based on a rolling standard deviation of returns, and also to a simple constant-volatility baseline.

Performance is measured using MAE, RMSE, MSE, and QLIKE. Overall, this setup is meant to test whether a GARCH-style model can capture time-varying volatility better than a simple baseline, while keeping the evaluation realistic and time ordered.

from pyspark.sql import functions as F
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

def run_garch_one_ticker_log_returns_only(
    mbo_spark_df,
    ticker="CL.c.0",
    start_date="2025-11-18",
    end_date="2026-01-20",
    freq="10s",
    single_session_only=None,
    min_points_per_session=30,
    train_frac=0.70,
    val_frac=0.15,
    mean_model="Zero",
    vol_model="GARCH",
    p=1,
    q=1,
    o=0,
    dist="normal",
    scale_returns=100.0,
    realized_vol_window=6,
    trading_days_per_year=252,
    trading_seconds_per_day=23 * 60 * 60,
    show_plots=True
):

    # data extraction
    df = (
        mbo_spark_df
        .filter(
            (F.col("symbol") == ticker) &
            (F.col("session_date") >= F.lit(start_date)) &
            (F.col("session_date") <= F.lit(end_date))
        )
        .select("session_date", "second_bucket", "filled_price")
        .where(F.col("second_bucket").isNotNull())
        .orderBy("session_date", "second_bucket")
    )

    if single_session_only is not None:
        df = df.filter(F.col("session_date") == F.lit(single_session_only))

    pdf = df.toPandas()

    if pdf.empty:
        raise ValueError(f"No data for {ticker} from {start_date} to {end_date}")

    pdf["session_date"] = pd.to_datetime(pdf["session_date"])
    pdf["second_bucket"] = pd.to_datetime(pdf["second_bucket"])

    print(f"Ticker: {ticker}")
    print(f"Raw rows: {len(pdf)}")
    print(f"Sessions: {sorted(pdf['session_date'].dt.strftime('%Y-%m-%d').unique().tolist())}")

    # resample each session
    session_frames = []

    for session, g in pdf.groupby("session_date"):
        g = g.sort_values("second_bucket").copy()
        g = g.dropna(subset=["filled_price"])

        if g.empty:
            continue

        g = g.set_index("second_bucket")
        s = g["filled_price"].resample(freq).last().ffill().dropna()

        if len(s) < min_points_per_session:
            continue

        tmp = s.to_frame(name="filled_price").reset_index()
        tmp["session_date"] = pd.to_datetime(session)
        session_frames.append(tmp)

    if not session_frames:
        raise ValueError("No usable sessions after resampling.")

    ts_df["log_return"] = ts_df.groupby("session_date")["log_price"].diff()
    ts_df = ts_df.dropna(subset=["log_return"]).copy()

    series_return = ts_df.set_index("second_bucket")["log_return"].copy()
    series_session = ts_df.set_index("second_bucket")["session_date"].copy()

    series_return = series_return[keep_mask]
    series_session = series_session[keep_mask]

    if len(series_return) < 200:

    print(f"Usable return observations: {len(series_return)}")

    n = len(series_return)

        plt.grid(True, alpha=0.3)

        plt.show()

        plt.show()
