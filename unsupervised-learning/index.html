#Overview

This notebook continues from the EDA, focusing on the unsupervised learning of the inherent structures within the identified features and targets (rv_next_60, rv_next_300). This part leverages on the Principal Component Analysis (PCA), T-distributed Stochastic Neighbor Embedding (t-SNE), Hierarchical Clustering, Cumulative Sum (CUSUM) statistics and Singular Spectrum Analysis (SSA). The plan is as follows.

  • Setup, load data, preprosessing and train/val/test spilt

  • Feature standardization

  • PCA on the multivariate time series, inspecting the global collinearity among the features, plot the cumulative explained variance and identify the top components.

  • t-SNE to inspect the non-linear clustering as suggested in the cross-symbol correlation from EDA and potential asset class clustering.

  • Hierarchical clustering and dendrogram to inspect the similarity among the symbols

  • CUSUM analysis to illustrate the daily change points, further zoom into the intraday change points by 5s resampled in the volatility expansion period

  • Apply SSA to the top features identified in PCA and targets (rv_next_60, rv_next_300) in train dataset, reconstruct the validation dataset, forecast and evaluate the test dataset.

CUSUM Statistics¶

The CUSUM (Cumulative Sum) statistic is often used to detect subtle shifts and change points in a process over time.

Here we used CUSUM to investigate the target rv_next_60 (a finer scale than ‘rv_next_300’) on a daily frequency first, identified the most volatile date and zoomed into the specific date on a 5-seconds resampling to identify the change points.

Daily CUSUM Statistics¶

# helper function

def cusum_daily(spark_df, symbol_to_analyze, feature_col, sample_fraction=0.5):

    """
    Performs daily resampled CUSUM analysis.

    Args:
        spark_df (pyspark.sql.DataFrame): The input Spark DataFrame (sdf_clean).
        symbol_to_analyze (str): The financial symbol to analyze.
        feature_col (str): The column to perform CUSUM analysis on.
        sample_fraction (float): Fraction of data to sample before converting to Pandas.

    Returns:
        tuple: (time_index, cusum_positive, cusum_negative, change_points, h_threshold, series_to_analyze)
    """
    single_symbol_sdf_cusum = (
        spark_df.filter(F.col('symbol') == symbol_to_analyze)
                .select(F.col('second_bucket').cast('string').alias('second_bucket_str'), feature_col)
                .orderBy('second_bucket_str')
    )


    single_symbol_sdf_cusum = single_symbol_sdf_cusum.sample(False, fraction=sample_fraction, seed=42)


    single_symbol_pd_cusum = single_symbol_sdf_cusum.toPandas()

    if single_symbol_pd_cusum.empty:
        return pd.Series(), np.array([]), np.array([]), [], 0.0, np.array([])

    single_symbol_pd_cusum['second_bucket'] = pd.to_datetime(single_symbol_pd_cusum['second_bucket_str'])
    single_symbol_pd_cusum = single_symbol_pd_cusum.drop(columns=['second_bucket_str'])

    single_symbol_pd_cusum_filtered = single_symbol_pd_cusum.dropna(subset=[feature_col])

    # Resample to daily frequency

    daily_resampled_df = (
        single_symbol_pd_cusum_filtered.set_index('second_bucket')
                                         .resample('D')[feature_col].sum()
                                         .reset_index()
    )
    daily_resampled_df = daily_resampled_df.dropna(subset=[feature_col])

    series_to_analyze = daily_resampled_df[feature_col].values

def cusum_intraday(spark_df, symbol_to_analyze, target_date_str, resample_freq, sample_fraction=1.0, feature_col='rv_next_60'):
    """
    Performs CUSUM analysis for intraday.

    Args:
        spark_df (pyspark.sql.DataFrame): The input Spark DataFrame (sdf_clean).
        symbol_to_analyze (str): The financial symbol to analyze.

        resample_freq (str): The resampling frequency (e.g., '5s', '1min', 'H', 'D').

        tuple: (time_index, cusum_positive, cusum_negative, change_points, h_threshold, series_to_analyze)

Page Matrix¶

Page Matrix takes a time series y(1),…,y(T)y(1), \dots, y(T)y(1),…,y(T) and shapes it into a matrix:

P(y,L)=[y(1)y(L+1)…y(T−L+1)y(2)y(L+2)…y(T−L+2)⋮⋮⋱⋮y(L)y(2L)…y(T)]P(y, L) = \begin{bmatrix} y(1) & y(L + 1) & \dots & y(T - L + 1) \\ y(2) & y(L + 2) & \dots & y(T - L + 2) \\ \vdots & \vdots & \ddots & \vdots \\ y(L) & y(2L) & \dots & y(T) \\ \end{bmatrix}P(y,L)=⎣⎡​y(1)y(2)⋮y(L)​y(L+1)y(L+2)⋮y(2L)​……⋱…​y(T−L+1)y(T−L+2)⋮y(T)​⎦⎤​

# Define the window length for SSA
L = 60

def page_matrix(df, L):
    """
    Creates a 'page matrix' (trajectory matrix) from a multi-variate time series DataFrame
    using a non-overlapping window.

    Args:
        df (pd.DataFrame): Input DataFrame where rows are time steps and columns are series.
                           Index should be datetime or sorted.
        L (int): Window length (number of time steps per window).

    Returns:
        tuple: (np.ndarray page_matrix, int actual_timesteps_used)
    """
    N = df.shape[0] # Total number of time steps
    K = df.shape[1]


    num_windows = N // L

    result_page_matrix = np.zeros((L, num_windows * K)) # Renamed local variable

    actual_timesteps_used = num_windows * L

    for k in range(K): #
        series_data = df.iloc[:actual_timesteps_used, k].values
        for j in range(num_windows):
            start_idx = j * L
            end_idx = start_idx + L
            result_page_matrix[:, j * K + k] = series_data[start_idx:end_idx] # Updated variable usage

    return result_page_matrix, actual_timesteps_used # Updated variable usage
def reconstruct_series(reconstructed_page_matrix, L, n_features, original_index, original_columns):
    """
    Reconstructs the original time series from the reconstructed page matrix.

    Args:
        reconstructed_page_matrix (np.ndarray): The matrix reconstructed from SVD components.
        L (int): Window length.
        n_features (int): Number of original series (columns in original DataFrame).
        original_index (pd.Index): The datetime index of the original DataFrame.
        original_columns (list): List of column names from the original DataFrame.

    Returns:
        pd.DataFrame: Reconstructed time series DataFrame.
    """
    if reconstructed_page_matrix.size == 0:
        return pd.DataFrame()

    num_rows_matrix, num_cols_matrix = reconstructed_page_matrix.shape
    num_windows = num_cols_matrix // n_features # Total number of "column" windows across all series

    reconstructed_length_per_series = num_windows * L

    reconstructed_data = {col: np.zeros(reconstructed_length_per_series) for col in original_columns}
    reconstructed_counts = {col: np.zeros(reconstructed_length_per_series) for col in original_columns}

    for feature_idx, col_name in enumerate(original_columns):
        series_matrix_cols = range(feature_idx, num_cols_matrix, n_features) # Columns in page_matrix corresponding to this series

        for window_idx, col_in_matrix in enumerate(series_matrix_cols):
            start_in_full_series = window_idx * L
            end_in_full_series = start_in_full_series + L

            reconstructed_data[col_name][start_in_full_series:end_in_full_series] += reconstructed_page_matrix[:, col_in_matrix]
            reconstructed_counts[col_name][start_in_full_series:end_in_full_series] += 1

    for col_name in original_columns:
        reconstructed_data[col_name] = np.divide(reconstructed_data[col_name], reconstructed_counts[col_name],
                                                out=np.zeros_like(reconstructed_data[col_name]),
                                                where=reconstructed_counts[col_name]!=0)

    reconstructed_df = pd.DataFrame(reconstructed_data, columns=original_columns)

    reconstructed_df.index = original_index[:reconstructed_length_per_series]

    return reconstructed_df

    for i in range(n_components_to_use):
        elementary_matrix = Sigma[i] * np.outer(U[:, i], Vt[i, :])
        reconstructed_page_matrix += elementary_matrix

    reconstructed_df = reconstruct_series(
        reconstructed_page_matrix,
        L,
        n_features,
        original_df_for_reconstruction.index,
        original_df_for_reconstruction.columns.tolist()
    )


    original_training_series[group_name] = original_df_for_reconstruction.iloc[:actual_timesteps_used]
    reconstructed_training_series[group_name] = reconstructed_df

    print(f"  Reconstruction for {group_name} training data complete. Reconstructed series shape: {reconstructed_df.shape}")

Takeaway¶

  1. Clustering Helps

Global / local, linear / non-linear clusterings help understand the similarites among the features.

  1. Features Extraction by Clusters

It might be more efficient and interpretable to extract the features based on the clustering to improve the model performance.

  1. Violent Volatility Expansion

At 60s or even shorter horizon,volatility is essentially a measure of the speed of price discovery within a single minute or less.

  1. SSA Results Differ among Asset classes

SSA works better in Crude Oil and US Treasury than EURO FX. It’s worthwhile to investigate into EURO FX inherent noise or additional exogenous variables.

Limitation¶

  1. Computation Complexity:

  • SVD: for a time series of length N and window L, the Page Matrix Embedding generates non-overlapping columns whereas Hankel Matrix Embedding generates the overlapping columns by sliding a window of length of L across the data one step at a time, the resulting computational complexity is O(NL)O(NL)O(NL) vs O(NL2)O(NL^2)O(NL2), so Page Matrix embedding was used here.

  • Lack of parallelism in t-SNE

  1. Matrix Embedding:

  • Though Page Matrix is more computational efficient, Hankel captures the “flow” of the time series, making it better for real-time filtering and smoothing.

  1. Hyperparameter Tuning:

  • Hierarchical Clustering ( features selected, distance metric )

  • SVD ( truncate thresholding )

  • Matrix Embedding ( window size)

This notebook did not focus on the hyperparameter tuning, we leverage on the domain knowledge as a starting point.

Application¶

  1. Clustering: both local (t-SNE) and hierarchical clustering could lead to discover the links among the trading instruments in terms of hedging, risk management and portfolio construction etc.

  2. CUSUM statistics might be useful in change points detection in terms of risk and market regime etc.

Further Studies¶

Some possible expansions from here:

  1. CUSUM Statistics can be further studied in the queue model, and we can analyze the stablility of the queue to recognize or possibly predict the market regime changes.

  2. Construct the Hankel Matrix instead of Page Matrix to better study the autocorrelation for modeling.

  3. SSA denoise can be integrated into the modeling in next steps and reiterate to finetune the hyperparameters such as the window size and SVD truncate threshold.
